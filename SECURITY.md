# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

1. Open a private security advisory in the repository, if enabled.
2. If private advisories are unavailable, email the repository maintainer directly.
3. Do not open public issues for suspected vulnerabilities.

Please include:

1. A clear description of the issue.
2. Steps to reproduce.
3. Affected versions or commit hashes.
4. Any proof-of-concept details needed to validate.

We will acknowledge reports as quickly as possible and coordinate remediation and disclosure timing.

## Scope

This policy covers the source code and official releases of this repository.
